Лабораторная работа
Программирование сетевых приложений

Защита последних лабораторных работ
1. Осуществляется в 3 этапа.
2. 1й этап: демонстрация базовой (без индивидуального задания) части приложения.
3. 2й этап: демонстрация индивидуального задания (выдается после демонстрации базой части).
4. 3й этап: анализ исходного кода.

демонстрация базовой части приложения
5. Запускается сервер и консоль, 4 разных клиента (не менее 2х компьютеров, лучше больше) работающих достаточно долго; таймер для сервисов должен быть взведен на 3 минуты (это параметр сервера).
6. Клиенты должны отыскать сервер: 1) с помощью широковещания (через позывной); 2) с помощью DNS-имени (студент должен прописать имя собственного сервера в зоне прямого просмотра DNS, DNS-имя -- параметр клиента).
7. Сервер, через три минуты должен отключить сервисы по таймауту.
8. Все подключения и отключения клиентов должны регистрироваться сервером в виде письма клиенту.
9. Запускается 4 разных клиента (не менее 2х компьютеров) работающих достаточно долго.
10. Выполнить команду stop.
11. Запустить 5й клиент. Он должен зависнуть.
12. Выполнить команду start. 5й клиент должен заработать.
13. Проверить команду statistics. Должно выдаваться общее количество подключений, количество активных подключений, завершившихся по таймауту и т.д. по книге.
14. Запустить 4 разных клиента (не менее 2х компьютеров) работающих достаточно долго.
15. Выполнить команду wait.
16. Запустить 5й клиент. Он должен зависнуть.
17. При завершении работы 4х ранее запущенных клиентов, должен заработать 5й.
18. Запускается 4 разных клиента (не менее 2х компьютеров) работающих достаточно долго.
19. Выполнить команду shutdown. Консоль должна стать недоступной для ввода команд.
20. 2 из 4х работающих клиентов завершить аварийно (снять), а 2 других должны доработать до таймаута.
21. После завершения всех клиентов сервер должен благополучно завершить свою работу.
22. Запустить 4 разных клиента (не менее 2х компьютеров) работающих достаточно долго.
23. Выполнить команду exit.
24. Консоль должна стать недоступной для ввода команд.
25. Клиенты должны завершиться с кодом ошибки.
26. Сервер должен благополучно завершить свою работу.

Примеры индивидуальных заданий
27. Администратор оправляет письмо серверу, в котором содержится расписание работы сервера. сервис RAND работает по с 8:00 -- 12:00, сервис TIME c 9:00 -- 10:00 и т.п. Сервер должен периодически (параметр сервера) считывать свою почту и корректировать свое расписание работы.
28. Консоль сервера поддерживает команду OPEN_ACCEPT XXXX, где XXXX -- номер порта. После этой команды сервер должен соединяться с клиентами по предыдущему порту (пусть он будет параметризуемым) и по-новому. Количество новых портов для подсоединения неограниченно. Команда CLOSE_ACCEPT XXXX должна остановить подключение к порту XXXX и освободить порт.
29. Консоль сервера поддерживает команду LOAD_LIB XXXXX, где XXXX -- имя DLL-библиотеки с новыми сервисами. После обработки этой команды сервер обеспечивает работу сервисов старой библиотеки и новой. Загружаемых библиотек может быть сколько угодно. Команда UNLOAD_LIB XXXX --выгружает библиотеку XXXX. Можно выгрузить все библиотеки. В этом случае сервер на все запросы сервиса должен отвечать отказом.
30. Администратор оправляет письмо серверу, в котором содержится расписание загрузки библиотек.
31. Разработка агента сервера.
32. Написать 2 сервера. Серверы должны обладать сервисом туннель, который транслирует пришедшую информацию на другой сервер. Другой сервер обрабатывает запрос и пересылает обратно на туннельный сервис. Туннельный сервис отправляет ответ клиенту.
